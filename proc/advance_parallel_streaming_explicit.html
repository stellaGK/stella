if flux tube simulation parallel streaming stays in ky,kx,z space with ky,kx,z local if full flux surface (flux annulus), will need to calculate in y space start the timer for the parallel streaming part of the time advance

allocate arrays needed for intermmediate calculations if simulating a full flux surface, will also need version of the above arrays that is Fourier transformed to y-space get (iv,imu,is) indices corresponding to ivmu super-index obtain (or -phi if driftkinetic_implicit=T) get d/dz, with z the parallel coordinate and store in dgphi_dz note that this should be a centered difference to avoid numerical unpleasantness to do with inexact cancellations in later velocity integration see appendix of the stella JCP 2019 for details

if driftkinetic_implicit=T, then only want to treat vpar . grad (-phi)*F0 term explicitly; in this case, zero out dg/dz term (or d(g/F)/dz for full-flux-surface) compute dg/dz in k-space and store in g0 if simulating a full flux surface, need to obtain the contribution from parallel streaming in y-space, so FFT d(g/F)/dz from ky to y transform d/dz (fully explicit) or d(-phi)/dz (if driftkinetic_implicit) from kalpha (ky) to alpha (y) space and store in g1y over-write g0y with d/dz (g/F) + Ze/T * d/dz (or -phi for driftkinetic_implicit).

multiply d(g/F)/dz and d/dz terms with vpa*(b . grad z) and add to source (RHS of GK equation) deallocate intermediate arrays used in this subroutine finish timing the subroutine
